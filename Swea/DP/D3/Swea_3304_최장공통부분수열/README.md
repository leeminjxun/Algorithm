# 🔁 SWEA: 3304 최장 공통 부분 수열 (DP , D3)

# 풀이과정

문자열 acaykp 와 capcak 가 주어졌을 때,
LCS 를 구하는 방식은 다음과 같다.

a -> {c, ca, cap, capc, capca, capcak}
ac -> {c, ca, cap, capc, capca, capcak}
aca -> {c, ca, cap, capc, capca, capcak}
acay -> {c, ca, cap, capc, capca, capcak}
acayk -> {c, ca, cap, capc, capca, capcak}
acaykp -> {c, ca, cap, capc, capca, capcak}

위의 방식으로 각 문자열의 index 를 늘려가며 비교를 한다.

무식하게 완전탐색으로 위와 같이 풀이 했을 경우, 시간 복잡도는 O(2 ^ N) 으로, N 의 최대길이가 1000 일 때 시간초과가 날 것이다.

[dp 풀이]

dp 를 정의했을 때, 인덱스의 정의는 다음과 같다.

dp[i][j]
i : S1 i 글자까지 LCS 비교
j : S2 j 글자까지 LCS 비교

즉, dp[i][j] 값은 S1 문자열의 처음부터 i - 1에 i 번째 문자를 추가하고, 
S2 문자열의 처음부터 j - 1 까지 에 j 번째 문자를 추가했을 때 완성 문자열들을 비교했을 때, 나타날 수 있는 최장 공통 부분 수열의 길이 이다.

[점화식]

문자열을 비교할 때, 경우의 수는 두가지 밖에 없다.

1. 현재 비교하는 문자가 같은 경우
해당 경우는 dp[i - 1][j - 1] 의 최장 공통 부분수열 에 + 1 을 해줘야한다.
-> dp[i][j] = dp[i - 1][j - 1] + 1
2. 현재 비교하는 문자가 다른 경우
해당 경우에서는 현재 비교하는 문자 중 i 를 빼거나, j 를 뺀 수열 중 더 큰 값을 선택한다.
-> dp[i][j] = Math.max(dp[i][j - 1], dp[i - 1][j])

로직 진행 순서를 시각적으로 표현하자면 다음과 같다.

비교 대상 : aba acb

[a]
1. dp[1][1] : a -> a
이건 같으므로 아무것도 없던 dp[0][0] 에서 각각 인덱스에 a 를 추가하여, dp[1][1] = 1;
2. dp[1][2] : a -> ac
현재 비교 문자는 c 이다. a 와 다르다. 다음 단계를 진행한다.
   - 1. dp[0][2] : a 를 뺴는 경우 {} -> ac
   - 2. dp[1][1] : c 를 빼는 경우 a -> a
    c 를 빼는 경우는 위에서 진행했던 dp[1][1] = 1 이므로 dp[1][2] = 1
3. dp[1][3] : a -> acb
비교 문자열이 다르다. 
   - 1. dp[0][3] -> 0
   - 2. dp[1][2] -> dp[1][1](2번 단계 참고) -> 1

[ab]
1. dp[2][1] : ab -> a
b 와 a 비교 시 같지 않다.
   - 1. dp[1][1] : b 를 빼는 경우 a -> a -> 1
   - 2. dp[2][0] : a 를 빼는 경우 ab -> {}
2. dp[2][2] : ab -> ac
   - 1. dp[1][2] : a -> ac
   - 2. dp[2][1] : ab -> a -> 1
3. dp[2][3] : ab -> acb
ab 와 acb 의 b 가 같다.
dp[1][3] : a -> ac 의 경우에서 각각 문자열 뒤에 d 를 추가한다.
dp[2][3] = dp[1][2] + 1; -> 2
