# 풀이과정

해당 문제에서 조건은 다음과 같다.

1. O 와 B 는 각각 1 에서 시작한다.
2. O 와 B 는 동시에 버튼을 누를 수 없다.
3. O 와 B 는 동시에 이동이 가능하며, 둘 중 하나의 로봇이 버튼을 누를 때 나머지 로봇은 이동이 가능하다.
4. O 와 B 가 버튼을 누르는 순서는 반드시 주어진 순열을 따라야한다.

[ 핵심 로직 ]

`int move = Math.abs(locO - target);
timeO = Math.max(totalTime, timeO + move) + 1;`

totalTime 은 이전의 로봇이 버튼을 누르기까지 걸린 시간이다.

move 는 해당 로봇이 현재 버튼에서 다음 버튼까지 이동하는데 걸린 시간이다.

만약 이전 로봇(B)이 버튼을 누르기까지 걸린 시간이 현재 로봇(O)이 다음 버튼으로 가는 시간보다 크다면,

B 가 버튼에 도달하기 전에 O 는 이미 목표 버튼에 도달할 수 밖에 없다.

if -> (B, 3) (O, 2)

(B, 3) 
loc_B  = 1
time_B = 0
total = 0;


move = |loc_B - 3| = 2

time_B = max(total, time_B + 2) + 1

total = time_B = 3

즉 첫번째 명령인 B 가 3번 버튼을 누르는데에는 이동하는 시간 2, 버튼을 누르는 시간 1 이 걸린다.

이 때 다음 명령은 (O, 2) 이다, 즉 O 가 2번 버튼을 누르는 경우이다.

하지만 B 가 이미 1칸 이동했을 때, 두 로봇은 동시에 이동이 가능하므로 O 또한 1칸 이동하였다.

이때 O 는 2번 버튼에 도착해 있고 첫번째 명령이 끝나길 기다리고 있다. -> "B 가 버튼에 도달하기 전에 O 는 이미 목표 버튼에 도달할 수 밖에 없다."

그러므로 (O, 2) 명령의 경우, O 가 이동하는 시간은 0 이고 버튼을 누르는 시간 1초만이 필요하다. -> time_O = total + 1
