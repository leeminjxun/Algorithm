# ❌ BOJ: 11000 강의실 배정 (Greedy / PriorityQueue, Gold4)

https://www.acmicpc.net/problem/11000

# 풀이 과정

김종혜 선생님한테는 Si에 시작해서 Ti에 끝나는 N개의 수업이 주어지는데, 최소의 강의실을 사용해서 모든 수업을 가능하게 해야 한다. 

S[i] = i 번째 수업이 시작하는 시각
T[i] = i 번째 수업이 끝나는 시각

수업이 끝난 직후에 다음 수업을 시작할 수 있다. (즉, Ti ≤ Sj 일 경우 i 수업과 j 수업은 같이 들을 수 있다.)

3
1 2
1 3
2 4
2 3
3 5

T[] = {1, 2, 3}
S[] = {3, 4, 5}

class 1 : 1 - 3 , 3 - 5
class 2 : 2 - 4

우리는 각각의 강의가 끝나는 시간을 저장해야한다.
하지만, 그 중 에서도 가장 먼저 끝나는 강의의 시간을 가장 앞 쪽에 배치해야한다.

현재 가장 먼저 끝나는 강의 의 시간보다 새로 시작하려는 강의 의 시간이 작다면,
새로운 강의실을 통해 수업을 진행하고 해당 강의 의 끝나는 시간을 또한 저장한다.

위와 반대로, 가장 먼저 끝나는 강의 시간이 새로 시작하려는 강의 시간과 같거나 그 보다 작다면,
해당 강의실에서 새로운 강의를 진행할 수 있다.

1. 수업 정보 (start, end)를 배열에 담고 start 기준 오름차순 정렬합니다.

2. 우선순위 큐(PQ)를 생성합니다. (강의실별 종료 시간을 저장)

3. 첫 번째 수업의 종료 시간을 PQ에 넣습니다. (강의실 1개 개설)

4. 두 번째 수업부터 차례대로 확인합니다:

   - (가장 빨리 끝나는 강의실 종료 시간) ≤ (현재 수업 시작 시간) 이라면:

     - 해당 강의실을 이어서 쓸 수 있습니다! PQ에서 기존 종료 시간을 꺼내고(poll), 현재 수업의 종료 시간을 새로 넣습니다(add).

   - 그렇지 않다면:

     - 이어서 쓸 수 있는 강의실이 없으므로 새로운 강의실을 하나 더 써야 합니다. 현재 수업의 종료 시간을 PQ에 그냥 넣습니다(add).

5. 모든 수업을 확인한 후 PQ의 크기가 바로 필요한 최소 강의실 개수입니다.
