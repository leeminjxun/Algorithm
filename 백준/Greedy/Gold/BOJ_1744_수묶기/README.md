# ✅ BOJ: 1744 수 묶기 (Greedy, Gold4)

https://www.acmicpc.net/problem/1744

# 풀이 과정

어떤 수열이 {0, 1, 2, 4, 3, 5}일 때, 
그냥 이 수열의 합을 구하면 0+1+2+4+3+5 = 15이다. 
하지만, 2와 3을 묶고, 4와 5를 묶게 되면, 0+1+(2*3)+(4*5) = 27이 되어 최대가 된다.

정렬을 통하여 가장 큰 수와 그 다음으로 큰 수 순서대로 묶어야한다.

[예외적 상황 1]

{-100, -10, 10, 100} 과 같은 수열이 있을 때,

100 부터 순서대로 곱한다면 100 * 10 + (-10 * -100) 을 하여 최댓값 2,000 이 나오게 된다.

하지만 

{-100, -10, 0, 10, 100}

위의 경우에 경우 이전의 최댓값과 동일하다. 하지만 위 로직을 그대로 사용한다면 1,000 - 100 = 900 이라는 값이 나오게 된다.

[해결 방법]

1. 우선 양수 인 값과 음수 인 값을 나눈다.

2. 0 은 양수와 곱하였을 때 최댓값이 나올 수 없으므로, 음수에 넣는다.

3. 양수는 내림차순, 음수는 오름차순 정렬한다.

4. 각각의 정렬된 값 2개를 꺼내 꺼내 곱해준다.
   - 하지만 양수의 경우, 두 수 중 하나의 값에라도 1이 있다면 곱했을 경우 최대가 되지 못한다.
   - ex) 1 * 2 = 2 but 1 + 2 = 3
   - 그러므로 해당 경우, 곱하지 않고 더해준다.

5. 만약 각각의 queue 값이 1 과 같다면 곱할 수 없으므로, 그냥 더해준다.
