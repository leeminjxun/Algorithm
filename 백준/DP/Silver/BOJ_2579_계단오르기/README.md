# ❌ BOJ: 2579 계단오르기 (DP, Silver3)

# https://www.acmicpc.net/problem/2579

## 백준 2579 - 계단 오르기 힌트

---

### 문제 핵심 조건 정리

```
1. 한 번에 1칸 또는 2칸 오를 수 있다.
2. 연속된 3칸을 연속으로 밟을 수 없다.
3. 마지막 계단(꼭대기)은 반드시 밟아야 한다.
```

---

### 생각의 출발점

n번째 계단에 도달하는 방법은 **딱 2가지 경로**밖에 없습니다.

```
경우 A: (n-1)번째 계단에서 1칸 올라온 경우
경우 B: (n-2)번째 계단에서 2칸 올라온 경우
```

여기서 **조건 2** 때문에 경우 A가 항상 가능한 게 아닙니다. 어떤 조건일 때 경우 A가 불가능한지 생각해보세요.

---

### 핵심

n번째 계단을 밟을 때 **"직전에 연속으로 몇 칸을 밟았는지"** 가 중요합니다.

```
n번째 계단을 밟는 상황
 ├── n-1번째도 밟고 온 경우  → n-2번째는 반드시 ?
 └── n-2번째에서 건너뛴 경우 → n-1번째는 반드시 ?
```

---

### dp 정의 힌트

단순히 `dp[n] = n번째 계단까지의 최대 점수`로 정의하면 **조건 2를 표현할 수 없습니다.**

n번째 계단에 도달했을 때 **직전 상태**가 무엇인지를 dp 정의에 포함시켜야 합니다.

```
dp[n][0] = n번째 계단이 연속 1번째로 밟힌 경우의 최대 점수
dp[n][1] = n번째 계단이 연속 2번째로 밟힌 경우의 최대 점수
```

---

### 점화식 도출 방향

각 상태로 도달하는 **직전 경로**가 무엇인지 생각해보세요.

```
dp[n][0] 이 되려면 → n-1번째를 밟지 않고 왔다는 뜻 → 직전 상태는 ?
dp[n][1] 이 되려면 → n-1번째를 밟고 왔다는 뜻     → 직전 상태는 ?
                                                   (단, 연속 3칸 금지)
```

최종 답은 `dp[n][0]`과 `dp[n][1]` 중 **더 큰 값**입니다.
